Search with RMMApi

This page describes the basic and advanced search capabilities of Resource metadata REST api. It is created as swagger-ui has some limitations. This page also explains various search options in details.

Use /RMMApi/search

Search type Resource endpoint Request Parameters Description
Search any phrase /RMMApi/search searchphrase=chemistry It searches the phrase throughout the mongo database and returns all the records which contain this phrase.
Operarion OR /RMMApi/search searchphrase=chemistry SRD 69 It searches given number of phrases throughout the mongo database and returns records with any of given words it finds in it. It is doing simple "OR" operation
Operation AND /RMMApi/search searchphrase="chemistry SRD 69" It searches given words in quotes in a record. It is a phrase search. Simple AND operation
Advanced OR /RMMApi/search searchphrase=chemistry "SRD 69" This option is for doing advanced OR for example you can OR between phrase and words.
Advanced AND /RMMApi/search searchphrase="chemistry" "SRD 69" "any other phrase" This will perform "AND" operation with given phrases in quotes and fine the records which will fulfill this request.

Use /RMMApi/advancedsearch

This serach is seperated only because of swagger limitation. /advancedsearch endpoint accepts any number of key=value pairs. Following are the example searches that /advancedsearch can perform.

Search type Resource endpoint Request Parameters Description
Search any phrase /RMMApi/advancedsearch key=value It seraches any key=value entry in POD schema. E.g advancedsearch?title=Enterprise
Operarion OR /RMMApi/advancedsearch key=value&key=value&logicalOp=OR This type of request performs OR operation between various fields in POD. For example /records/advancedsearch?title=enterprise&keyword=data&logicalOp=OR this will look any entries/records which contains either title=Enterprise or keyword=data or both.
Operation AND /RMMApi/advancedsearch key=value&key=value&logicalOp=AND This type of request performs OR operation between various fields in POD. For example /records/advancedsearch?title=enterprise&keyword=data&logicalOp=AND this will look any entries/records which contains both title=Enterprise or keyword=data .
